Редактирование магазина

Здесь отображаются текущие данные магазина, чтобы администратор мог в дальнейшем обновлять адрес, название и переводить точку в неактивный статус без удаления истории.

Вернуться к магазинам
Экран ожидает, что backend положит в модель атрибут editShopReq.

Что уже подключено

  • GET /admin/shops/edit/{id} открывает экран редактирования.
  • Форма предзаполняется через editShopReq.
  • POST /admin/shops/edit должен принять shopId, название и адрес магазина.

Следующий шаг

После обновления формы можно добавить отдельное действие для мягкой деактивации точки.
