Редактирование сотрудника

Страница показывает текущие данные сотрудника и подготавливает форму для будущего обновления без физического удаления записи из системы.

Вернуться к сотрудникам
Текущий экран ориентируется на модель editEmployeeReq и список shops.

Что уже подключено

  • GET /admin/employees/edit/{id} открывает экран редактирования.
  • Форма предзаполняется через editEmployeeReq.
  • Список магазинов передается в модель атрибутом shops.
  • POST /admin/employees/edit должен принять employeeId, основные поля сотрудника и новое значение магазина.

Следующий шаг

После сохранения можно добавить success-message и отдельный сценарий мягкого увольнения сотрудника.
