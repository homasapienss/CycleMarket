Основной сценарий

Для менеджера backend берет магазин из текущего сотрудника. Для администратора магазин выбирается на этой странице, а затем используется для перехода к операциям с остатками и заказами.

Операционный контур точки

Менеджер сразу работает со своей точкой. Администратор сначала выбирает магазин, а затем переходит к нужному разделу.

Admin

Выбор рабочей точки

Выберите магазин один раз, а затем откройте нужный раздел. Тот же `shopId` уйдет и в заказы, и в остатки.

Для административного сценария backend должен передать список магазинов в атрибуте shops.

Действия для выбранной точки

После выбора магазина администратор открывает тот же операционный контур точки, но уже в явно выбранном контексте.

Orders

Управление заказами

Открывает контур заказов. Если магазин выбран, `shopId` уйдет в запрос; если нет, backend сам решит общий сценарий.

Stock

Управление остатками

Открывает остатки. Если магазин выбран, он уйдет в `shopId`; если нет, backend сам обработает общий сценарий.

Один и тот же выбор магазина используется для обоих переходов. Если магазин не выбран, форма отправит запрос без `shopId`.

Ограничения роли

Эти правила можно использовать как ориентир для backend-проверок и навигации.

Менеджер работает только с остатками своей точки.
Администратор сначала выбирает магазин, а затем открывает операции уже для выбранной точки.
Менеджер не должен менять свои учетные данные и привязку к магазину.
Создание и управление сотрудниками остается в зоне ответственности администратора.
