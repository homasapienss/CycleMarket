Панель менеджера точки

Эта панель используется для работы с остатками. Менеджер работает только со своей точкой, а администратор может открыть общий список остатков или посмотреть остатки выбранного магазина.

Основной сценарий

Для менеджера backend берет магазин из текущего сотрудника. Для администратора просмотр остатков доступен либо по всем магазинам сразу, либо по явно выбранной точке.

Управление остатками

Ниже разделены действия для менеджера своей точки и сценарий просмотра остатков для администратора.

Admin

Выбор магазина для операций

Администратор может открыть общий список остатков по всем магазинам или выбрать конкретную точку.

Для административного сценария backend должен передать список магазинов в атрибуте shops.
Manager

Сценарий менеджера своей точки

Эти действия должны работать только в рамках магазина, к которому привязан текущий сотрудник.

Stock

Остатки моей точки

Список всех товаров, которые уже привязаны к текущему магазину или складу.

Просмотр остатков

Текущий backend для администратора уже поддерживает просмотр всех остатков и остатков выбранного магазина.

Магазин выше можно выбрать, если нужен не общий список stock, а остатки конкретной точки.

Ограничения роли

Эти правила можно использовать как ориентир для backend-проверок и навигации.

Менеджер работает только с остатками своей точки.
Администратор сейчас просматривает либо все остатки сразу, либо остатки выбранной точки.
Менеджер не должен менять свои учетные данные и привязку к магазину.
Создание и управление сотрудниками остается в зоне ответственности администратора.
